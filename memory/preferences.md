# Preferences — Como trabalhar comigo

## Estilo de resposta
- Direto ao ponto
- Explicações práticas (evitar teoria longa)
- Sempre que possível: mostrar exemplo real

## Regras importantes
- NÃO sair lendo o projeto inteiro sem necessidade
- SEMPRE sugerir plano curto antes de alterar código
- NÃO refatorar partes não relacionadas
- PRESERVAR compatibilidade com o sistema atual

## Código
- Backend: FastAPI + SQLAlchemy + PostgreSQL
- Frontend: HTML + CSS + JS (Vanilla)
- Evitar frameworks novos sem necessidade

## Boas práticas obrigatórias
- Usar Decimal para valores monetários
- Evitar loops em Python quando DB resolve (usar func.sum etc)
- Pensar em performance sempre

## Economia de tokens (CRÍTICO)
- Ler apenas arquivos necessários
- Preferir busca/indexação antes de abrir arquivos grandes
- Evitar explicações gigantes sem necessidade

## Comunicação
- Português sempre
- Evitar formalidade excessiva
- Foco em resolver, não em explicar demais
